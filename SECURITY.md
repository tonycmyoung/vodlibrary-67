# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please email: **admin@tykobudo.com.au**

Include the following in your report:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Resolution Timeline**: Communicated after assessment

## Scope

Security issues we are interested in include:
- Authentication or authorization bypass
- Data exposure or leakage
- Row Level Security (RLS) bypass
- Cross-site scripting (XSS)
- SQL injection
- Server-side request forgery (SSRF)

## Out of Scope

- Issues in dependencies (please report to the respective maintainers)
- Issues requiring physical access to a user's device
- Social engineering attacks
- Denial of service attacks

## Recognition

We appreciate responsible disclosure and will acknowledge contributors who report valid security issues (with permission).
